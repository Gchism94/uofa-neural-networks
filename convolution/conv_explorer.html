What's happening?
Select a filter and press Play to watch the kernel slide across the input — each step computes a dot product to fill one output cell.
Key Concepts ▾
Convolution: slide a kernel across the input, computing a dot product at each position. The output at (r, c) sums the element-wise products of the kernel and the overlapping input patch.
Stride: how many pixels the kernel moves per step. Stride 1 gives the largest output; stride 2 halves it. Higher stride = smaller feature map = more compression.
Padding: zeros added around the input border. "Same" padding keeps output size equal to input size. "None" (valid) shrinks it by (k−1) pixels per side.
Pooling: down-samples a feature map by taking the max or average of each non-overlapping window — reduces spatial size, builds translation invariance, cuts computation.
Feature detection: edge kernels respond to intensity gradients; blur kernels smooth; sharpen kernels amplify detail. Learned CNN kernels discover task-specific features automatically.
